Coverage & blind spots

This GO-WITH-FIXES verdict gates the static / DC correctness of the design. It is NOT evidence about the dimensions below — these are not covered by any block and (mostly) cannot be settled before fabrication. Bench-gate them on the populated board using the TP3-TP8 commissioning test points.

What this verdict is NOT evidence about (gate blind spots)

RX audio AC/dynamic performance — end-to-end RX gain, in-band (300Hz-3.4kHz) frequency response, ADC noise floor / input-referred SNR through the 22k/3.3k divider
The RX block proves only the abs-max ceiling (gong stays below AVDD+0.3); nothing checks the floor — that quiet door-station speech clears the ADC noise floor with workable SNR after the −18dB attenuation. DESIGN.md defers this to bench ('trim 22k/3.3k against measured ADC full-scale'). Not derivable pre-fab.
TX audio delivered level & source-impedance budget at line 3 (OUTP→R26 2.2k→C14→R28 2.2k→P3 into the TV20/S talk-amp)
TX block proves only OUTP abs-max + idle high-Z. No min/max TX-level requirement exists, so 'too quiet to be heard' and 'overdriving the TV20/S amp' (AUDIO-6) are both un-gated; the ~4.4kΩ source-Z divider into the amp is unquantified, and TX-out-reach is itself an explicit DESIGN.md open bench item.
Bus-coupled hum / ground-loop in-band noise from the deliberate P1↔GND bond (SAFE-3 isolation not met)
The galvanic barrier is intentionally dropped; DESIGN.md lists 'Hum with the P1↔GND bond once RX is live' as an open bench item with no number. The supply-integrity block is AVDD-side, not signal-CMRR/hum-side. This is the failure mode the abandoned isolation transformer used to prevent — only a populated-board measurement can settle it.
WiFi-TX-burst cross-rail coupling onto the shared +3V3 plane feeding ES8311 PVDD/DVDD, and into the I2S/MCLK clock domain
The supply block owns AVDD only (clean LP5907+FB1). PVDD/DVDD deliberately stay on the +3V3 plane carrying the ~350mA WiFi burst (~90mV modeled droop). No block checks whether that hash modulates the codec digital section, MCLK/BCLK/LRCK timing, or folds into converted audio. In-band and unexamined.
I2S/MCLK clock integrity & audio-sample jitter at U3, and I2C link margin at the 0.127mm fab-limit / 0.40mm-pitch routing
No block addresses dynamic timing of the audio data/clock bus (setup/hold, jitter); the clock group was landed on the JTAG pads and the bus runs at JLCPCB's fab-limit clearance. Clean digital audio transport has no signal-integrity check.
Regulator thermal / junction temperature under sustained WiFi-TX + audio load (U2 SGM2212 ~0.44W peak in SOT-223; U4 LP5907)
The power block checks dropout HEADROOM (rail stays in regulation) but not junction temperature / dissipation; no thermal block exists and the heatsink copper area was not verified against the load. Refuter estimated TJ~103C<150C but it is unconfirmed.
ES8311 intra-rail power-up skew (Note 3: VDDP/VDDD within 10ms) and absolute VMID voltage
AVDD is gated by LP5907 EN=+3V3 so ordering (VDDD-on-when-VDDA-on) is satisfied, but whether the LDO soft-start keeps the skew under 10ms depends on unspecified turn-on times. VMID's numeric value is not printed in the ES8311 datasheet, so RX absolute headroom rests on an assumed VMID≈AVDD/2 (swing-relative-to-VMID is firm).
PCB-physical correctness beyond schematic — actual layer stackup / ground-plane handling of the AGND-vs-GND star point, copper-thieving slivers, via-in-pad, and the SPPJ322300 internal NC/NO switch sense
This review was netlist + datasheet + measured-bus-envelope only; it did not open doorbell.kicad_pcb. Star-ground implementation, trace routing, and the WF26-core switch make/break logic (SW3/SW4 unreadable from datasheet resolution) can only be judged on the layout / bench, not from connectivity.
